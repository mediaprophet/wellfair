# Draft Definition File: Sanctuary Mode Specification
**Application:** wellfare
**Module:** Sanctuary Mode (Highly Privileged Environment)
**Purpose:** To define a distinct, highly privileged environment within the application, completely isolated from standard health tracking interfaces, designed to provide inviolable protection for the user's most sensitive data.

---

## 1. Authentication and Initialization
Access to Sanctuary Mode requires a deliberate and secure transition from the standard app environment.

*   **The Threshold (Authentication):** Accessing this mode requires a secondary, specific PIN or biometric unlock that is entirely distinct from the decoy vault.
*   **Visual Shift:** The interface should immediately shift to a "Dark Mode" or a distinct color palette, providing immediate visual feedback that the user is now operating within their most sensitive data layer.

---

## 2. Core Menu Structure and Functions

### 2.1 The Unvarnished Log (Data Entry)
This is the primary ingestion point for the messy, complex reality of the user's life.

*   **Log a Veiled Assertion:** A function to record definitive, highly sensitive facts, such as hidden substance use or undisclosed accidents.
*   **Log a Hypothesis Node:** A specialized entry form for epistemic uncertainty, allowing the user to record suspicions or symptoms (like suspected poisoning) without clinically or legally committing to the claim.
*   **Immutable Anchoring:** A background function that automatically hashes these entries and commits them to a Decentralized Ledger (DLT) to establish an Asymmetric Provenance Anchor.

### 2.2 The Tripwire Dashboard (Alert Management)
This section is where the user interacts with the system's active protective measures.

*   **Active Opaque Collisions:** A feed showing external queries (e.g., a doctor attempting to prescribe a contraindicated medication) that the Prolog engine has successfully intercepted.
*   **Resolution Engine:** An interface allowing the user to review what external query tripped the wire, see the predicted consequences, and choose an action: Keep Hidden, Reveal Temporarily, or Export to Doctor.
*   **Internal Sentinel Alerts:** Private notifications triggered when the user's logged data strongly correlates with threat-modeling ontologies, such as patterns of coercive control.

### 2.3 The Synthesis Engine (Threat & Logic Analysis)
This menu exposes the analytical power of the local Prolog engine to the user, acting as their internal detective.

*   **Contradiction Audits:** A tool to input external claims or fabricated assertions (like a false police report) and run them against the user's historically immutable record.
*   **Incoherence Reports:** The output generated by the Prolog engine, mathematically proving structural or temporal impossibilities in an adversary's claims.
*   **Sentinel Ruleset Management:** A secure directory where the user can view or update the standard threat-modeling ontologies guiding the system's background analysis.

### 2.4 Evidentiary Export & Defense (Actionable Output)
When the user needs to leverage their hidden truth to defend themselves, this section manages the secure extraction of data.

*   **Isolate Defense Node:** A function allowing the user to select specific sub-graphs of their timeline, health logs, and Veiled Assertions.
*   **Generate Verifiable Presentation (VP):** The tool that packages the selected RDF-Star triples, original raw assets, and DLT hashes into a cryptographically sealed package linked to the user's DID.
*   **Ephemeral VPN Serving:** The mechanism to generate a temporary, secure link so a judge, lawyer, or investigator can locally verify the cryptographic signatures in a standard desktop browser.

### 2.5 Contingency Protocols (Emergency Escrow)
This is the final fail-safe for extreme circumstances where the user fears for their physical safety or freedom.

*   **Duress Configuration:** The settings area to manage the decoy vault PIN, ensuring the attacker cannot mathematically prove the Sanctuary Mode even exists.
*   **Dead Man's Switch:** The configuration tool for time-locked releases.
*   **Nymtech Routing:** The system that automatically routes the encrypted RDF graph to pre-designated trusted nodes (journalists, lawyers) if the user fails to check in.
